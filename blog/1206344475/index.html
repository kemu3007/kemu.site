Djangoプロジェクトをカスタムテンプレートから作成する

Django レポジトリテンプレート

GitHubのレポジトリテンプレート機能を利用し、Django Project Templateを作成したのでその説明と導入手順です。

テンプレートからレポジトリを作成する。

Django Project Template

画面右上 Use this template ボタンをクリックし、新規レポジトリを作成することでテンプレートを利用したレポジトリが作成されます。

テンプレートの内容

.github/workflows, Makefile

mainにpush, pull_requestをcreate / syncしたタイミングで自動的にCIを発火するようにしています。

CIではMakefileに定義しているlint, test コマンドを実行しています。Djangoデフォルトのテストフレームワークを利用する形で定義しているのでpytestに切り替える場合はMakefile内のtestコマンドを修正することで自動的に適用されます。

linterの設定内容は setup.cfg, pyproject.toml を確認してください。

black, isortがバージョン問題でフォーマット結果が変わるタイミングがあったりするのでrequirements.dev.txtでバージョン管理を行なっています。

docker-compose.yml

python:3.10イメージをベースにしたrequirements.txtの内容をインストールするようにしたカスタムイメージを立ち上げ、python src/manage.py runserver を実行しています。

動作確認

docker-compose v2を利用しています。

zsh $ docker compose up -d デフォルトで8000番ポートで起動するようにしているのでブラウザでlocalhost:8000にアクセスし、以下の画面が表示されればOKです。
