Python Typehint new / deprecated

発行日時: 2022.06.20 01:47

最終更新日: 2022.06.29 02:04

気がつかないうちに追加 / 非推奨になっていた項目があったため紹介します。非推奨項目が多いのでよく利用されるもののみ記載しています。

詳しくは公式ドキュメントを参照してください。

document: https://docs.python.org/ja/3/library/typing.html

追加されたTypehint

利用するケースが想定され、便利そうなもののみ記載しています。詳しく知りたい方は公式ドキュメントを参照してください。

typing.Literal

Python3.8~ 利用可能

持ちうる具体的な値を指定することが可能です。

def test_function(value: Literal[1, 2, 3])
    ...


test_function(0) <-- raise Error

typing.TypedDict

Python3.8~ 利用可能

from typing import TypedDict

dict_type = TypedDict("DictType", { "key": str, "value": str })

class DictType(TypedDict):
    key: str
    value: str

Python3.11~ はRequired, NotRequired型が追加されます。

class DictType(TypedDict):
    key: str
    value: str
    optional_key: NotRequired[str]

NotRequiredはキーが存在するか存在しないかわからない場合に利用することができます。

Requiredはtotal=Falseと指定したケースについても必須なキーに対して利用することで型の付与が可能です。

3.10~ は is_typeddict() も追加され、判定を行うことが可能です。(ちょっと利用するケースがわからないけれど）

typing.TypeGuard

Python3.10~ 利用可能

def is_str(val: str | float) -> TypeGuard[str]:
    return isinstance(val, str)

判定式でTrueが返却された際の引数の型を指定可能です。

typing.overload

引数と戻り値の型が連動している際に利用できます。

@overload
def test_function(value: int) -> int
    ...

@overload
def test_function(value: List[int]) -> List[int]
    ...

def test_function(value: int | List[int]) -> int | List[int]
    ...


test_function(1) --> int
test_function([1, 2, 3]) -> list[int]

typing.final

Python3.8~ 利用可能

継承、上書きされたくないケースで利用できます。

class BaseClass:
    @final
    def test_function(self):
        ...

class ExtendClass(BaseClass):
    def test_function <-- raise ERROR

@final
class NotBaseClass:
    ...

class ExtendClass2(NotBaseClass) <-- raise ERROR

非推奨 Typehint

他にも非推奨項目が存在しますが良く利用されているのはこの3つなので割愛します。詳しくは公式ドキュメントを参照してください。

typing.Dict

Python3.9~よりdeprecated

組み込みのdictがdict[str, str]といった型に対応しました。

typing.List

Python3.9~よりdeprecated

組み込みのlistがlist[str]といった型指定に対応しました。

typing.Callable

Python3.9~よりdeprecated

collections.abc.CallableがCallable[[str], str]といった型指定に対応しました。
