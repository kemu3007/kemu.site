Django JWT認証を利用する

導入

  • インストール

zsh $ pip install djangorestframework djangorestframework-simplejwt

  • settings.py

python INSTALLED_APPS = [ ... "rest_framework", "rest_framework_simplejwt", ] REST_FRAMEWORK = { ... "DEFAULT_AUTHENTICATION_CLASSES": ( ... "rest_framework_simplejwt.authentication.JWTAuthentication", ) }

  • urls.py

    • 基本的なAPI Viewに関しては提供されているためとりあえず導入
    • TokenObtainPairView : トークンの生成
    • TokenRefreshView : トークンの再発行
    • TokenVerifyView : トークンの有効性確認
    • 他にもblacklistなど用意されています。

```python from django.urls import include, path from rest_framework_simplejwt.views import TokenObtainPairView, TokenRefreshView, TokenVerifyView

jwt_urls = [ path("", TokenObtainPairView.as_view()), path("refresh/", TokenRefreshView.as_view()), path("verify/", TokenVerifyView.as_view()), ]

urlpatterns = [ path("token/", include(jwt_urls)), ] ```

    • refresh はアクセストークンを再発行するためのトークン

トークンの有効性確認

トークンを token/verify/ にPOST

  • レスポンス
  • 200 OKのみ返却される
  • アクセストークン、リフレッシュトークンどちらも有効性確認可能
  • デフォルトではトークンがリフレッシュされた場合でもリフレッシュ前のトークンが有効期限内であれば利用可能
    • BLACKLIST_AFTER_ROTATION をTrueにする必要がある
  • デフォルトでは一つのユーザに紐づくトークンを複数作成できるためセキュリティ的な面で調整が必要そう
    • やり方は要確認 ( 設定でありそうな気がする )

json {}

トークンを付与してログインが必要なページにアクセスする

  • curl

bash $ curl localhost:8080/api/v1/user/ -H "Authorization: Bearer{accesskey}"

  • rest_framework.test.APIClient

python client = APIClient().credentials(HTTP_AUTHORIZATION=f"Bearer {accesskey}") client.get("localhost:8080/api/v1/user/")

トークンが有効であれば200、不正であれば401が返却されます。

終わりに

トークン認証はCORSを気にしなくて良く使いやすい反面ちょっとセキュリティ的に怖いなと感じた。 フロント側でどこにトークンを持つべきなんだろうかHTTP Only Cookie? LocalStorage/SessionStorage?

トークンが盗まれるケースを前提において既存のトークン無効化する機能/有効期限を短く設定しておくといった対応が必要になる。
