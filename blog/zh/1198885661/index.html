视觉化OpenApi文件。

発行日時: 2022.04.10 05:29

最終更新日: 2022.04.22 10:14

OpenApi文件。

当一个RESTful API被创建时，一个OpenApi文件被生成并传递给前端端，以自动生成接口，并且在最近的开发中使用类型安全开发。

OpenApi文件以yaml或json格式输出，其原始形式不适合人类阅读，但可以通过使用Swagger / Redoc等工具将其可视化，作为文档使用。

##如何使用

在GitLab的情况下，只需将文件名swagger.yml放入仓库即可获得Swagger- UI视图。

参考资料: https://docs.gitlab.com/ee/api/openapi/openapi_interactive.html

在Redoc格式的情况下，可以通过使用以下符号生成HTML文件，因此可以考虑使用案例，如在CI中创建文件后在GitLab / GitHub Pages上发布。

$ redoc-cli bundle openapi.yml

我个人觉得Swagger- UI比Redoc好，因为Swagger- UI在屏幕上有更多的信息，而且URLs更容易理解。

如果你想检查从OpenApi文件生成的Swagger- UI/Redoc，你可以通过以下工具进行检查

OpenApi Viewer | kemu tools
