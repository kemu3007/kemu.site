使用Gmail API推送通知

発行日時: 2022.03.06 07:47

最終更新日: 2022.03.06 03:19

它允许将该过程与电子邮件挂钩，并与处理过程穿插进行。

###环境。

  • Python 3.10
  • google- api- python- client==2.36.0
  • google_auth_oauthlib=0.4.6
  • google- cloud- pubsub==2.9.0

###程序。
①从GCP控制台创建一个项目并激活Gmail API

(2) 点击Gmail API - > 认证信息 - > 创建认证信息，并创建一个OAuth客户端ID（选择桌面应用很方便，因为如果你只想随手玩玩，就不用考虑重定向URI的问题）。

(iii) 创建一个OAuth客户端ID后，点击访问并下载JSON。

(iv) 由于下载的JSON没有被赋予令牌，执行以下代码，点击控制台显示的URL并同意，将生成的代码粘贴到控制台，并以pickle的形式保存认证信息。(该描述是基于djnago的，但没有必要意识到这一点)。

import json
import pickle
from pathlib import Path

from django.conf import settings
from django.core.management.base import BaseCommand
from google_auth_oauthlib.flow import InstalledAppFlow


class Command(BaseCommand):
    def handle(self, *args, **options):
        credential_path: Path = settings.BASE_DIR.parent / "auth.json"
        credential_scopes = ["https://mail.google.com/"]
        raw_credentials = json.loads(credential_path.read_text())
        flow = InstalledAppFlow.from_client_config(raw_credentials, credential_scopes)
        credentials = flow.run_console()
        with open("token.pickle", "wb") as token:
            pickle.dump(credentials, token)

  1. 创建一个pub/sub，这是GCP控制台的Gmail推送通知所需要的。首先创建主题，这时你不需要勾选任何复选框。

  2. 为该主题设置PUBLISHER。在屏幕右侧信息面板的权限部分点击 "添加负责人"，添加 "gmail- api- push@system.gserviceaccount.com "作为 "Pub/Sub Publisher "角色。

(vii) 创建一个订阅。选择 "拉动 "作为交付类型。该主题将是步骤5中创建的主题。

  1. 根据第4步创建的认证信息，运行users.watch。在这种情况下，通过指定labelIds为INBOX，通知控制了事件的触发。
from googleapiclient.discovery import build

with open(settings.BASE_DIR / "mail/token.pickle", "rb") as token:
    credentials = pickle.load(token)
service = build("gmail", "v1", credentials=self.credentials)
# settings.PUBSUB_TOPICには  projects/{project_name}-{project_id}/topics/{topic_name} のような値が入ります。
service.users().watch(userId="me", body={"labelIds": ["INBOX"], "topicName": settings.PUBSUB_TOPIC}).execute()

(ix) 设置回调函数和订阅。

from google.cloud.pubsub_v1 import SubscriberClient

# settings.PUBSUB_SUBSCRIPTIONにはprojects/{project_name}-{project_id}/subscriptions/{topic_name} のような値が入ります
SubscriberClient(credentials=credentials).subscribe(settings.PUBSUB_SUBSCRIPTION, callback=callback)

⑩通过订阅获得的数据格式如下。
https://developers.google.com/gmail/api/guides/push#receiving_notifications

请注意，本例中的 "messageId "与gmail ID无关。

(11) 根据historyId获取信息。

如果使用步骤10中获得的historyId，由于使用startHistoryId查询的方便，可能不会获得。因此，有必要使用之前的值，如果不存在，则使用user.watch时的historyId。

histories = service.users().history().list(userId="me", historyTypes=["messageAdded"], labelId="INBOX", startHistoryId=past_history_id).execute(num_retries=5)
message_id = histories["history"][0]["messagesAdded"][0]["message"]["id"]
message = self.service.users().messages().get(userId="me", id=message_id).execute(num_retries=5)

⑫由于在第12步中检索到了信息，所以通知slack，保存到DB，或做任何其他你喜欢的处理。

该文本以base64编码，应该由base64.urlsafe_b64decode或类似的方法处理。

###注意。

  • 请不要将您的证书上传到如github，因为它涉及到GCP的访问权。
  • user.watch需要定期运行。如果能把它变成一个命令，并通过oferia或cron定期执行，那就更好了。

###一般评论。

有很多更简单的方法可以将邮件发送至slack本身，但我认为如果你想穿插其他进程或想将邮件信息存储在一个易于使用的DB中，这将是非常有用的。

我们经历了一段艰难的时期，因为周围很少有日语文件，但我们在一天之内成功地将它变成了一个可以工作的slackbot，所以我对此感到高兴。
