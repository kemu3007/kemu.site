Python Typehint new / deprecated

発行日時: 2022.06.20 01:47

最終更新日: 2022.06.20 01:47

:
有些项目在没有被注意到的情况下被添加/废弃，在此介绍。由于有许多被废弃的项目，所以只列出了最经常使用的项目。

更多信息请参见官方文件。

文件：https://docs.python.org/ja/3/library/typing.html

##添加了类型提示。

只有那些可能被使用和有用的才会被列出。欲了解更多信息，请参考官方文件。

＃＃＃键入.Literal

Python3.8~可用

可以指定可以持有的具体数值。

def test_function(value: Literal[1, 2, 3])
    ...


test_function(0) <-- raise Error

typing.TypedDict。

Python3.8~可用

from typing import TypedDict

dict_type = TypedDict("DictType", { "key": str, "value": str })

class DictType(TypedDict):
    key: str
    value: str

Python3.11~ 增加了 Required, NotRequired 类型。

class DictType(TypedDict):
    key: str
    value: str
    optional_key: NotRequired[str]

当不知道钥匙是否存在时，可以使用NotRequired。

Required也可以用于强制键，为指定total=False的情况下分配一个类型。

3.10~还增加了is_typeddict()，它可以做出决定。(虽然我不确定它将被如何使用......)

###键入.TypeGuard。

Python 3.10~可在

def is_str(val: str | float) -> TypeGuard[str]:
    return isinstance(val, str)

可以在决策表达式中指定返回True时的参数类型。

#＃＃键入.overload

当参数和返回类型相连时，可以使用这个方法。

@overload
def test_function(value: int) -> int
    ...

@overload
def test_function(value: List[int]) -> List[int]
    ...

def test_function(value: int | List[int]) -> int | List[int]
    ...


test_function(1) --> int
test_function([1, 2, 3]) -> list[int]

###打字.最后

Python3.8~可用

这可以在不希望继承或覆盖的情况下使用。

class BaseClass:
    @final
    def test_function(self):
        ...

class ExtendClass(BaseClass):
    def test_function <-- raise ERROR

@final
class NotBaseClass:
    ...

class ExtendClass2(NotBaseClass) <-- raise ERROR

废弃的类型提示

其他废弃的项目也存在，但由于它们是最常用的三个项目，所以被省略了。更多信息请参见官方文件。

###打字.Dict。

从Python 3.9~开始被弃用。

内置的dict现在支持dict[str, str]等类型。

###键入.列表。

从Python 3.9~开始被弃用。

内置的列表现在支持类型规格，如list[str]。

###打字.可调用。

从Python 3.9~开始被弃用。

Collections.abc.Callable现在支持Callable[[str], str]这样的类型规范。
