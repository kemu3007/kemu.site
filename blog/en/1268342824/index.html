Linking AWS CodePipeline and GitHub to push images to ECR

発行日時: 2022.06.11 06:19

最終更新日: 2022.06.12 09:33

We used GitHub OAuth authentication, CodePipeline integration, and Auto Push to ECR using CodeBuild, and we will show you how to do it.

By using OAuth authentication, developers no longer need to manually manage tokens to AWS, and authorization management after CodeBuild is completely tokenless because the AWS framework can be used.

GitHub and AWS CodePipeline integration

Use OAuth authentication.

You can create a connection from CodePipeline > Connections > Create Connection.

We used GitHub OAuth authentication, CodePipeline integration, and Auto Push to ECR using CodeBuild, and we will show you how to do it.

GitHub and AWS CodePipeline integration

Use OAuth authentication.

You can create a connection from CodePipeline > Connections > Create Connection.

Prepare Elastic Container Registory, CodeBuild

Initialize ECR and create CodeBuild first, as it will be needed to set up CodePipeline.

Elastic Container Registory

Create a repository from Amazon ECR > Create Repository. In this case, we will assume a Private image.

CodeBuild Settings

CodeBuild > Create Project.

This time, we will grant privileges to build the Docker image. Also, please use the buildspec file.
Other settings such as image and log management can be set as desired.

Configure IAM settings.

Create a CodePipeline

CodePipeline > Create Pipeline.


Based on the connection you just created, select the repositories you want to use and the branches on which you want to make releases.

CodeBuild Settings

CodeBuild > Create Project.

This time, we will grant privileges to build the Docker image. Also, please use the buildspec file.
Other settings such as image and log management can be set as desired.

Configure IAM settings.

IAM > Roles > codebuild- *****

Roles created/used in CodeBuild must be granted privileges to ECR.

In this operation check, we granted AmazonEC2ContainerRegistryFullAccess from Add Permissions > Attach Policy.

Create a CodePipeline

CodePipeline > Create Pipeline.


Based on the connections, CodeBuild, and ECR you just created, select the repositories you want to use and the branches on which you want to make releases.

We will skip the deployment step this time.

buildspec.yml

Create a buildspec.yml in the following form and create it under the repository.

version: 0.2
phases:
  pre_build:
    commands:
      - aws ecr get-login-password --region ap-northeast-1 | docker login --username AWS --password-stdin *****.dkr.ecr.ap-northeast-1.amazonaws.com
  build:
    commands:
      - docker build -t container-image . 
  post_build:
    commands:
      - docker tag container-image *****.dkr.ecr.ap-northeast-1.amazonaws.com/container-image:latest
      - docker tag container-image *****.dkr.ecr.ap-northeast-1.amazonaws.com/container-image:$CODEBUILD_RESOLVED_SOURCE_VERSION
      - docker push -a *****.dkr.ecr.ap-northeast-1.amazonaws.com/container-image

Available environment variables such as $CODEBUILD_RESOLVED_SOURCE_VERSION can be found on the following page.

[Environment Variables in Build Environment](https://docs.aws.amazon.com/ja_jp/codebuild/latest/userguide/build- env- ref- env- vars.html)

Auto Push is performed to ECR by the above settings. In the next article, we will deploy the pushed ECR image to ECS.
