Using Django JWT Authentication

発行日時: 2022.04.21 11:40

最終更新日: 2022.04.22 09:48

Introduction

  • Installation
$ pip install djangorestframework djangorestframework-simplejwt
  • settings.py
INSTALLED_APPS = [
    ...
    "rest_framework",
    "rest_framework_simplejwt",
]
REST_FRAMEWORK = {
    ...
    "DEFAULT_AUTHENTICATION_CLASSES": (
        ...
        "rest_framework_simplejwt.authentication.JWTAuthentication",
    )
}

  • urls.py

  • Basic API View is provided, so it is introduced for the time being.

  • TokenObtainPairView : Generate a token.

  • TokenRefreshView : Refresh the token.

  • TokenVerifyView : Verify token validity.

  • Other services are also available, such as blacklist.

from django.urls import include, path
from rest_framework_simplejwt.views import TokenObtainPairView, TokenRefreshView, TokenVerifyView

jwt_urls = [
    path("", TokenObtainPairView.as_view()),
    path("refresh/", TokenRefreshView.as_view()),
    path("verify/", TokenVerifyView.as_view()),
]

urlpatterns = [
    path("token/", include(jwt_urls)),
]

Check operation

Issuing Tokens

Create a user as appropriate and POST username and password to token/.

  • response
  • access is a token used as Authorization: Bearer {accesskey} in the header when using the API.
  • refresh is a token to reissue an access token

Check token validity

POST a token to token/verify/.

  • response
  • 200 OK only returned
  • Validity can be checked for both access and refresh tokens
  • By default, tokens can be used even if they are refreshed, as long as the tokens before the refresh are still valid
    - BLACKLIST_AFTER_ROTATION needs to be set to True
  • Security aspects may need to be adjusted as multiple tokens can be created that are tied to a single user by default.
    - How to do this needs to be confirmed (I think it might be in the settings).
{}

Grant a token to access pages that require login

  • curl.
$ curl localhost:8080/api/v1/user/ -H "Authorization: Bearer{accesskey}"
  • rest_framework.test.APIClient.
client = APIClient().credentials(HTTP_AUTHORIZATION=f"Bearer {accesskey}")
client.get("localhost:8080/api/v1/user/")

If the token is valid, 200 is returned; if invalid, 401 is returned.

End

Token authentication is easy to use without worrying about CORS, but at the same time, it is a bit scary from a security standpoint.
Where should we have tokens on the front side, HTTP Only Cookie? LocalStorage/SessionStorage?

It is necessary to set a function to invalidate existing tokens and set a short expiration date on the assumption that the tokens may be stolen.
