Create a health care application with GitHub Actions and GitHub Pages

発行日時: 2022.04.17 03:44

最終更新日: 2022.04.26 03:33

Original concept

  • I want to be able to check it online on any device, but I don't want to set up an EC2 or something and deploy it.
  • Data is entered once a day.
  • The data to be registered are minimum blood pressure, maximum blood pressure, pulse rate, pulse pressure, and body weight.
  • Daily data is displayed in easy- to- understand charts.

Application Specification

  • Data is entered once a day using GitHub Actions.
  • Save the data in JSON format on GitHub Actions and push to the repository.
  • The distribution will use GitHub Pages and visualize the data with Chart.js.

Application Configuration

Enter data in GitHub Actions

You can put any value into GitHub Actions in the following yml format.

name: write-data

on:
  workflow_dispatch:
    inputs:
      blood_min:
        default: 0
        required: true
      blood_max:
        default: 0
        required: true
      heart_rate:
        default: 0
        required: true
      heart_pulse:
        default: 0
        required: true
      weight:
        default: 0
        required: true

The actual UI is as follows

Input values are available in the form {{ github.event.inputs.blood_min }}.

Save data in JSON format

To make the data easier to read in JavaScript, the values passed to GitHub Actions are processed in a Python script. (The sample code uses ZoneInfo, so Python 3.9 or later must be used.)

import argparse
from datetime import datetime
import json
from pathlib import Path
from zoneinfo import ZoneInfo


parser = argparse.ArgumentParser()

parser.add_argument("blood_min", type=int)
parser.add_argument("blood_max", type=int)
parser.add_argument("heart_rate", type=int)
parser.add_argument("heart_pulse", type=int)
parser.add_argument("weight", type=float)


blood_json = Path("docs/blood.json")
weight_json = Path("docs/weight.json")


if __name__ == "__main__":
    today = datetime.now(ZoneInfo("Asia/Tokyo")).date()
    arguments = parser.parse_args()
    blood = json.loads(blood_json.read_text())
    blood[str(today)] = {
        "min": arguments.blood_min,
        "max": arguments.blood_max,
        "heart_rate": arguments.heart_rate,
        "heart_pulse": arguments.heart_pulse,
    }
    blood_json.write_text(json.dumps(blood))
    weight = json.loads(weight_json.read_text())
    weight[str(today)] = {"weight": arguments.weight}
    weight_json.write_text(json.dumps(weight))

This time, to distribute docs/ in the repository main branch, prepare docs/blood.json and docs/weight.json and store the JSON data.

Now you are ready to deliver a JSON file with date and body data.

Visualize JSON data

Since JSON as it is is inconvenient for humans to read, it is visualized in a graph format.

The library used is Chart.js.

index.html 

<!DOCTYPE html>
<html lang="ja">
  <head>
    <title>Health Logger</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-EVSTQN3/azprG1Anm3QDgpJLIm9Nao0Yz1ztcQTwFspd3yD65VohhpuuCOmLASjC" crossorigin="anonymous">
  </head>
  <body>
    <div class="container mt-5">
      <h2>Blood</h2>
      <canvas id="blood" height="80px"></canvas>
      <h2>Weight</h2>
      <canvas id="weight" height="80px"></canvas>
    </div>
    <script src="https://cdn.jsdelivr.net/npm/chart.js@3.7.1/dist/chart.min.js"></script>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/js/bootstrap.bundle.min.js" integrity="sha384-MrcW6ZMFYlzcLA8Nl+NtUVF0sA7MsXsP1UyJoMp4YLEuNSfAP+JcXn/tWtIaxVXM" crossorigin="anonymous"></script>
    <script>
      const bloodCtx = document.getElementById("blood").getContext('2d');
      fetch("blood.json").then(response => {
        response.json().then(blood => {
          console.log(blood)
          const bloodChart = new Chart(bloodCtx, {
            type: 'line',
            data: {
              datasets: [
                {
                  label: "最高血圧",
                  data: Object.values(blood).map(item => item.max),
                  borderColor: 'rgb(255, 0, 0)'
                },
                {
                  label: "最低血圧",
                  data: Object.values(blood).map(item => item.min),
                  borderColor: 'rgb(0, 0, 255)'
                },
                {
                  label: "脈拍",
                  data: Object.values(blood).map(item => item.heart_rate),
                  borderColor: 'rgb(255, 0, 255)'
                },
                {
                  label: "脈圧",
                  data: Object.values(blood).map(item => item.heart_pulse),
                  borderColor: 'rgb(128, 128, 128)'
                },
              ],
              labels: Object.keys(blood)
            }
          })
        })
      })
      const weightCtx = document.getElementById("weight").getContext('2d');
      fetch("weight.json").then(response => {
        response.json().then(weight => {
          const weightChart = new Chart(weightCtx, {
            type: 'line',
            data: {
              datasets: [{
                label: "体重",
                data: Object.values(weight).map(item => item.weight),
                borderColor: 'rgb(0, 0, 0)'
              }],
              labels: Object.keys(weight)
            }
          })
        })
      })
    </script>
  </body>
</html>

Fetch JSON files that exist in the same hierarchy with fetch. In this case, you need to give the following command line arguments because it will fail with CORS if you run it locally.

- - allow- file- access- from- files.

For mac OS, please use the following command to quit Chrome and then start it. (We have not tested this argument on Windows.)

Chrome version: 100.0.4896.127

$ open "/Applications/Google Chrome.app" --args --allow-file-access-from-files

Problem

If you use GitHub Pages, you must be a GitHub Enterprise user to grant view restrictions on GitHub Pages.

Therefore, since the information will be made public to the entire world, it is necessary to take measures such as complicating URLs, not uploading elements that are tied to individuals other than numbers, and adding a noindex meta to prevent Google from indexing the information.
