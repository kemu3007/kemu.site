When you place the site on the command line, it is a common configuration such as setting an SSL certificate in the load balancer and specifying EC2 in the target group.

In this configuration, add CNAME records to the domain DNS record and write the DNS name of the load balancer.

However, if you want to set a load balancer for Apex Domain ( kemu.site for this site), you can not add CNAME records.

The IP of the load balancer is variable, and it is not recommended to act that may be changed due to restart, and there is a way to fix the IP of the load balancer, but it is costly.

If you are not currently using Route 53, you need to create a host zone in Route 53 and set the value of the NS record in that domain from the name dot com or Google Domain management screen. (It takes a little time to reflect changes.) It was reflected in about an hour. )
