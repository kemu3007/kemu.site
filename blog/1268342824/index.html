AWS CodePipelineとGitHubを連携させてECRにイメージをPushする

発行日時: 2022.06.11 06:19

最終更新日: 2022.06.12 09:33

GitHubのOAuth認証を利用し、CodePipelineと連携、CodeBuildを用いたECRへのAuto Pushを行ったのでやり方を紹介します。

OAuth認証を利用することで、デベロッパーはAWSへのトークンを手動で管理する必要がなくなり、CodeBuild以降の権限管理はAWSのフレームワークを利用できるため完全にトークンレスになります。

GitHubとAWS CodePipeline連携

OAuth認証を利用します。

CodePipeline > 接続 > 接続を作成 から連携を作成することができます。

Elastic Container Registory, CodeBuildの用意

CodePipelineの設定に必要になるため、先にECRの初期設定、CodeBuildの作成を行います。

Elastic Container Registory

Amazon ECR > レポジトリを作成 からレポジトリを作成します。今回はPrivateイメージを想定します。

GitHubとAWS CodePipeline連携

OAuth認証を利用します。

CodePipeline > 接続 > 接続を作成 から連携を作成することができます。

Elastic Container Registory, CodeBuildの用意

CodePipelineの設定に必要になるため、先にECRの初期設定、CodeBuildの作成を行います。

Elastic Container Registory

Amazon ECR > レポジトリを作成 からレポジトリを作成します。今回はPrivateイメージを想定します。

CodeBuildの設定

CodeBuild > プロジェクトの作成

今回はDockerイメージの構築を行うため、特権付与を行います。また、buildspecファイルを利用する形にしてください。
その他、イメージ, ログの管理などに関しては好きなように設定してOKです。

IAMの設定を行う

CodePipelineの作成

CodePipeline　> パイプラインを作成する


先ほど作成した接続に基づいて、利用するレポジトリ、リリースを行うブランチについて選択します。

CodeBuildの設定

CodeBuild > プロジェクトの作成

今回はDockerイメージの構築を行うため、特権付与を行います。また、buildspecファイルを利用する形にしてください。
その他、イメージ, ログの管理などに関しては好きなように設定してOKです。

IAMの設定を行う

IAM > ロール > codebuild-*****

CodeBuildで作成した/利用しているロールにECRへの権限を付与する必要があります。

今回の動作確認ではアクセス許可を追加 > ポリシーをアタッチ からAmazonEC2ContainerRegistryFullAccessを付与しました。

CodePipelineの作成

CodePipeline　> パイプラインを作成する


先ほど作成した接続, CodeBuild, ECRに基づいて、利用するレポジトリ、リリースを行うブランチについて選択します。

デプロイステップに関しては今回スキップします。

buildspec.yml

buildspec.ymlを以下のような形で作成し、レポジトリ配下に作成します。

version: 0.2
phases:
  pre_build:
    commands:
      - aws ecr get-login-password --region ap-northeast-1 | docker login --username AWS --password-stdin *****.dkr.ecr.ap-northeast-1.amazonaws.com
  build:
    commands:
      - docker build -t container-image . 
  post_build:
    commands:
      - docker tag container-image *****.dkr.ecr.ap-northeast-1.amazonaws.com/container-image:latest
      - docker tag container-image *****.dkr.ecr.ap-northeast-1.amazonaws.com/container-image:$CODEBUILD_RESOLVED_SOURCE_VERSION
      - docker push -a *****.dkr.ecr.ap-northeast-1.amazonaws.com/container-image

$CODEBUILD_RESOLVED_SOURCE_VERSION などの利用可能な環境変数は以下のページから確認できます。

ビルド環境の環境変数

上記設定によりECRにAuto Pushされます。次回はPushしたECRイメージからECSへのデプロイを行います。
