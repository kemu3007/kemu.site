AWS Apex Domainとうまく付き合う

サイトをAWS上に置いている場合、ロードバランサにSSL証明書を置き、ターゲットグループにEC2を指定するといった構成がよくある構成として挙げられます。

この構成の場合、ドメインのDNSレコードに通常CNAMEレコードを追加し、ロードバランサのDNS名を記載します。

しかし、Apex Domain(このサイトであれば kemu.site)に対してロードバランサを設定したい場合CNAMEレコードを追加することができません。

ロードバランサのIPは可変であり、再起動などにより変更される可能性が存在する行為は推奨されておらず、ロードバランサのIPを固定する方法もあるにはあるのですが費用がかかります。

AWS Global Accelerator

そのため、Route53のAレコードエイリアスの機能を利用します。

以上の形でエンドポイントのエイリアスを設定可能です。

もし現在Route53を利用していない場合はRoute53でホストゾーンを作成し、そのドメインのNSレコードの値をNSとするようにお名前ドットコムやGoogle Domainの管理画面から設定する必要があります。(変更の反映には少し時間がかかります。この時は確か1時間くらいで反映された。)

これにより可変なロードバランサのエンドポイントをApex Domainに指定することができました。

実際に登録される値としては2件のレコードが登録されていました。

```zsh $ dig kemu.site

; <<>> DiG 9.10.6 <<>> kemu.site ;; global options: +cmd ;; Got answer: ;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 7482 ;; flags: qr rd ra; QUERY: 1, ANSWER: 2, AUTHORITY: 0, ADDITIONAL: 1

;; OPT PSEUDOSECTION: ; EDNS: version: 0, flags:; udp: 512 ;; QUESTION SECTION: ;kemu.site. IN A

;; ANSWER SECTION: kemu.site. 60 IN A 54.178.253.199 kemu.site. 60 IN A 13.231.15.68

;; Query time: 103 msec ;; SERVER: 2404:1a8:7f01:b::3#53(2404:1a8:7f01:b::3) ;; WHEN: Tue Mar 22 00:36:06 JST 2022 ;; MSG SIZE rcvd: 70 ```

内部的にどのようにIPを割り振っているのかは少しわからないのですがこのように設定すればいいようにAレコードを挿入してルーティングしてくれるようです。

※IPアドレスによる直リンを防ぐのであればAWS WAFを利用する必要があります。（ろくな情報を置いていないので何も設定していないですが）
