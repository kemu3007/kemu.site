DRF 紐づくモデルの表示件数を制限する

djangoにはprefetch_related という機能が存在しており以下のように記載することで発行されるSQLを抑制することができます。

python Article.objects.prefetch_related("tags")

と記載することでManyToManyの値をキャッシュに保存し、SQLの発行数が減少します。

この機能を応用し、

python class ArticleViewSets(ModelViewSet): def get_queryset(self): return Article.objects.all().prefetch_related(Prefetch("tags", Tag.objects.filter(enable=True)))

のように記載することで返却されるarticle.tagsの値をフィルタリングすることが可能です。

しかし、このやり方はArticleのステータスが下書きの際のみtagsの中身を空に、公開の際にはすべてのtagsを公開するといったレコードに基づいた表示の変更は行うことができません。

その際に利用するのがprefetch_related_objectsです。

prefetch_related_objectを利用して上述のような処理を書くと下記のようになります。

python class ArticleViewSets(ModelViewSet): def retrieve(self, request, *args, **kwargs): instance: Article = self.get_object() tags = Tag.objects.all() # ステータスに応じて返却するファイルを制限する if instance.status == Article.StatusChoices.DRAFT: tags = Tag.objects.none() prefetch_related_objects([instance], Prefetch("tags", tags )) serializer = self.get_serializer(instance) return Response(serializer.data)

このように記載することで返却するtagsの値について制限をかけることが可能です。
