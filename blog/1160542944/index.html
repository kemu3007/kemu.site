Django4.0がリリースされたので適当に試してみる

release notes

依存ライブラリ

  • Django 3.2 asgiref==3.4.1 pytz==2021.3 sqlparse==0.4.2
  • Django4.0 asgiref==3.4.1 backports.zoneinfo==0.2.1 sqlparse==0.4.2 pytzに依存しなくなり、標準のzoneinfoモジュールに変更された形になります。 backports.zoneinfoは3.9以前をサポートするための依存です。

backports.zoneinfo

UniqueConstraint Expression

UniqueConstraintを設定する際にdesc(), asc()を指定してインデックスを張ることができるようになりました。

```python class TestModel(models.Model): code_1 = models.CharField(max_length=32) code_2 = models.CharField(max_length=32)

class Meta:
    constraints = [
        UniqueConstraint(
            Lower('code_1'),
            Lower('code_2').desc(),
            name='code_1_2_unique',
        ),
    ]

```

発行されるSQL sql CREATE UNIQUE INDEX "code_1_2_unique" ON "common_testmodel" ((LOWER("code_1")), (LOWER("code_2")) DESC);

QuerySet.contains

QuerySetの中に該当のモデルが存在するかチェックすることができます。

python TestModel.objects.contains(TestModel(id=1)) sql SELECT (1) AS "a" FROM "common_testmodel" WHERE "common_testmodel"."id" = 1 LIMIT 1

実行されるクエリはexists()と変わらないようです。（この程度の処理であれば）
