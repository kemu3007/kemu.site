OpenApiドキュメントを可視化する

OpenApiドキュメント

RESTful APIを作成した際にOpenApiファイルを生成しフロントエンド側に渡すことで自動でinterfaceを生成し、型安全に開発を行うということが最近の開発では利用されています。

OpenApiファイルはyamlもしくはjson形式で出力され、そのままの形式では人間が読むことに適していませんが、Swagger / Redocといったツールを利用して可視化することでドキュメントとしても利用することが可能です。

Swagger-UI

Redoc

利用方法

GitLabの場合、ファイル名をswagger.ymlとしてレポジトリに置くだけでSwagger-UI表示がされます。

参考: https://docs.gitlab.com/ee/api/openapi/openapi_interactive.html

Redoc形式にする場合は以下のように表記することでHTMLファイルを生成することができるため、CIで作成後GitLab / GitHub Pagesで公開することでドキュメントを公開するといったユースケースが考えられます。

zsh $ redoc-cli bundle openapi.yml

個人的にはRedocよりもSwagger-UIの方が画面内の情報量が多く、URLがわかりやすいため使うのであればSwagger-UIかなという気がしています。
