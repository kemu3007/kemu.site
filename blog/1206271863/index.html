GitHub ActionsとGitHub Pagesで健康管理アプリを作成する

発行日時: 2022.04.17 03:44

最終更新日: 2022.04.26 03:33

元々の構想

  • オンラインでどの端末でも確認できる形式にしたいがEC2などを立ててデプロイまではちょっとしたくない。
  • データの入力は一日一回。
  • 登録するデータは最低血圧, 最高血圧, 脈拍, 脈圧, 体重。
  • 日々のデータをチャートでわかりやすく表示。

アプリケーション仕様

  • データは一日一回 GitHub Actionsを利用して入力する。
  • GitHub Actions上でデータをJSON形式で保存し、レポジトリにpushする。
  • 配信はGitHub Pagesを利用し、Chart.jsでデータを可視化する。

アプリケーション構成

GitHub Actionsでデータを入力する

以下のyml形式でGitHub Actionsに任意の値を入れることができます。

name: write-data

on:
  workflow_dispatch:
    inputs:
      blood_min:
        default: 0
        required: true
      blood_max:
        default: 0
        required: true
      heart_rate:
        default: 0
        required: true
      heart_pulse:
        default: 0
        required: true
      weight:
        default: 0
        required: true

実際のUIは以下の通り

入力値は {{ github.event.inputs.blood_min }} という形式で利用することができます。

データをJSON形式で保存する

データをJavaScriptで読みやすくするため、GitHub Actionsに渡した値をPythonスクリプトで加工します。(サンプルコードはZoneInfoを利用しているため、Python3.9以降を利用する必要があります。)

import argparse
from datetime import datetime
import json
from pathlib import Path
from zoneinfo import ZoneInfo


parser = argparse.ArgumentParser()

parser.add_argument("blood_min", type=int)
parser.add_argument("blood_max", type=int)
parser.add_argument("heart_rate", type=int)
parser.add_argument("heart_pulse", type=int)
parser.add_argument("weight", type=float)


blood_json = Path("docs/blood.json")
weight_json = Path("docs/weight.json")


if __name__ == "__main__":
    today = datetime.now(ZoneInfo("Asia/Tokyo")).date()
    arguments = parser.parse_args()
    blood = json.loads(blood_json.read_text())
    blood[str(today)] = {
        "min": arguments.blood_min,
        "max": arguments.blood_max,
        "heart_rate": arguments.heart_rate,
        "heart_pulse": arguments.heart_pulse,
    }
    blood_json.write_text(json.dumps(blood))
    weight = json.loads(weight_json.read_text())
    weight[str(today)] = {"weight": arguments.weight}
    weight_json.write_text(json.dumps(weight))

今回はレポジトリmainブランチの docs/ を配信するため、docs/blood.json, docs/weight.json を用意し、JSONデータを保存します。

これにより、日付と身体データを持ったJSONファイルを配信する準備ができました。

JSONデータを可視化する

JSONそのままの形式では人間が読む際に不都合があるため、グラフ形式に可視化します。

利用したライブラリはChart.jsです。

index.html

同一階層に存在しているJSONファイルを fetch で取得します。この際、ローカルで動かす場合CORSで失敗するため、以下のコマンドライン引数を与える必要があります。

--allow-file-access-from-files

mac OSの場合は以下のコマンドでChromeを終了させてから起動させてください。(Windowsでこの引数の動作確認は行なっていません。)

動作確認を行ったChrome version: 100.0.4896.127

$ open "/Applications/Google Chrome.app" --args --allow-file-access-from-files

問題点

GitHub Pagesを利用する場合、GitHub EnterpriseでなければGitHub Pagesに対する閲覧制限を付与することができません。

そのため、全世界に公開することになってしまうため、URLの煩雑化、数値以外の個人と紐付ける要素をアップロードしない、Googleにインデックス登録されないようにnoindexメタを追加するなどといった対応が必要です。

あとモバイルアプリ版からActions起動ができたらいいなという思いがちょっとあります。（現状だとアプリ版だとActions起動できない&Web版はUIが最適化されていないので...API叩くのもトークン管理が面倒なのでなんとかしてくれないかなあ）
