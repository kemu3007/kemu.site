GitHub ActionとPagesでブログを作る

発行日時: 2022.03.06 04:25

最終更新日: 2022.03.06 11:44

環境

  • npm v8.3.1
  • node: v16.14.0
  • Angular: v13.2.5
  • @ng-bootstrap/ng-bootstrap: 12.0.0

※ npm run ng add @ng-bootstrap/ng-bootstrap が失敗する場合は一旦 npm config set legacy-peer-deps true としてから行なってください。

動作概要

行なったことは以下の通りです。 .github/workflows/deploy.yml あたりを見てもらえればわかるんじゃないかなと思います。

  1. GitHubのissueが作成/更新されたタイミングでGitHub Actionを走らせarticleラベルのついているissueを収集
import requests
import json
from pathlib import Path

if __name__ == "__main__":
    Path("articles/list.json").unlink(missing_ok=True)
    res = requests.get("https://api.github.com/repos/kemu3007/portal/issues?labels=article")
    articles = json.loads(res.content)
    with open("articles/list.json", "x") as f:
        f.write(json.dumps(articles))
  1. 収集されたissueをjson形式で保存し、Angularが記事を表示する際に参照するように設定
  2. ビルドしたAngularプロジェクトをGitHub Pagesで配信

Octokitを利用してGitHub Rest APIを利用することも考えたのですが手元で遊んでいる際にレート制限になってしまい面倒だったのでこの形式にしました。

デプロイには angular-cli-ghpages を利用しました。/ 以外でリダイレクト処理を行なった際に404 -> 目的のパスへの遷移を自動的に行なってくれます。--base-href, --cname のどちらかの引数を利用する必要があるのでそこだけ忘れないようにしてください。

  • 初期のドメインを利用する場合はbasehrefを指定(--base-href={repo_name})
  • カスタムドメインを利用する場合はcnameを指定(--cname=portal.kemu.site)

httpsの証明書も自動で発行してくれるためあまり気にする必要もありません。(カスタムドメインを利用する場合はCNAMEレコードを作成して{user_id}/github/ioに向ける必要があります。)

ブログのためだけにEC2借りてバックエンドとフロントエンド用意するのはちょっとなーとなっていたのでしばらくこれで行こうと思います。
