Angular 状態管理を行う @ngneat/elf

[@datorama/akita](https://github.com/datorama/akita)のページに[@ngneat/elf](https://github.com/ngneat/elf)を確認してねという以下の記載があったので動作確認 `Elf, a newer state management solution, has been published. We recommend checking it out 🚀` --- #### インストール 今回はセッションストレージにデータを保存するため `@ngneat/elf-persist-state` もインストールします。 ```zsh $ npm i @ngneat/elf @ngneat/elf-persist-state ``` --- #### 実装 akitaと異なり、cliを利用してファイルを生成して利用する際にinjectする形ではなく、以下の記述のみで動かすことが可能です。 - `shared/page.repository.ts` ```ts import { createStore, withProps } from '@ngneat/elf'; import { persistState, sessionStorageStrategy } from '@ngneat/elf-persist-state'; interface state { location: number; } const pageStore = createStore({ name: 'pageStore' }, withProps({ location: 0 })); persistState(pageStore, { key: 'page', storage: sessionStorageStrategy }); export class PageState { location = pageStore.getValue().location; update(key: keyof state, value: any) { pageStore.update((state) => ({ ...state, [key]: value, })); } } ``` --- #### 感想 akitaの構造(service, store, query)がわかりやすい代わりに複雑であるのに対してelfについては手軽で扱いやすいという印象を受けました。 今度httpリクエスト回りに関してもテストを行う予定です。 elfに完全移行してしまうのかと今までのプロジェクトの移行についてちょっと辛いかもしれないなと思ったのですが[この辺り](https://github.com/datorama/akita/discussions/770#discussioncomment-1620967)を見る限りでは維持はされるようです。
