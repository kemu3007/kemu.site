Google Adsenseで詰まった話、サイト構成など

発行日時: 2022.03.06 05:51

最終更新日: 2022.04.04 05:16

このサイトの構成は

お名前ドットコム -> Route 53 -> AWS ALB -(http1.1)> AWS EC2

という形になっています。実装当初は

お名前ドットコム -> AWS ALB -(http2.0)> AWS EC2

となっていたのですがadsenseに登録する&ALBの仕様上こうなりました。

adsenseに登録するうえで2点ほど躓くポイントがあったのでメモを残しておきます。

1. adsenseに登録しようとするとサイトにリーチできませんというエラーが出る

結論から言うとHttp Error 464が発生していました。

リクエストしていたhttpとロードバランサ内部でのプロトコルが一致せずエラーを吐いていたようです。

ALBのターゲットグループへの通信をhttp1.1にし、EC2上のnginxについてもhttp1.1に修正する必要がありました。

google search consoleでサイトマップを食わせた際に464エラーが画面に表示されようやくわかりました。

対策 

curl http://example.com/ --http1.1
curl http://example.com/ --http2

など手元でcurlコマンドを使って確認する必要があります。

2. お名前ドットコムはサブドメインにしかCNAMEレコードを登録できない

ALBにはIPアドレスが割り振られていますがそのIPアドレスは固定ではなくスケールアウトしたタイミング等で変化します。

そのためDNS名が割り振られておりドメインのDNS設定にCNAMEレコードを作成する必要があります。

しかし、お名前ドットコムはサブドメイン以外にCNAMEレコードの作成を許可していません。そこでお名前ドットコムが全体的につかいにくいなという思いもあったのでRoute53にドメインを移管しようと思ったのですがそれはできませんでした。

Route53で管理できるトップドメインに制限があり、siteはその対象外でした。

Amazon Route 53 に登録できる最上位ドメイン

そこでRoute53でゾーンを作成、作成したゾーン記載のNSをお名前ドットコム側でNSに設定し、Route53からALBに向けるという修正を行いました。

※ AWS Global Acceleratorを利用してIPアドレスを割り振り、利用するという方法もありますが、今回はお金をかけたくない&結構値段がしそうなので見送りました。
パフォーマンスが向上するようなのでそのあたりがシビアなプロジェクトであれば入れてもいいのかな～と思います。

AWS Global Accelerator

サイトにリーチできませんというエラーでググるとはてなブログ有料版の情報(www.にリダイレクト設定を行うもの)ばかり出てきてしまい、原因の解明に時間がかかりましたが解決してよかったです。
