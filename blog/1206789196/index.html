Graphene Django Graphql ドキュメントを生成する

発行日時: 2022.04.18 07:59

最終更新日: 2022.04.25 01:58

graphene-djangoを利用しスキーマの出力、スキーマからドキュメントの生成を行います。

環境 

Django==3.2.13
graphene-django==2.15.0

Queryの作成

スキーマを出力するために簡易的なQueryを作成します。具体的な導入手順等はgraphene-djangoのtutorialを行なってください。

出力内容が分かりやすくなるように各所にdescription, deprecation_reasonを記載しています。

import graphene
from django.contrib.auth.models import User
from django.contrib.contenttypes.models import ContentType
from graphene_django import DjangoObjectType


class UserType(DjangoObjectType):
    class Meta:
        model = User
        fields = "__all__"
        description = "Django Default User"


class ContenttypeType(DjangoObjectType):
    class Meta:
        model = ContentType
        fields = "__all__"
        description = "Django Contenttype"


class Query(graphene.ObjectType):
    users = graphene.List(UserType, description="ユーザ情報一覧取得API", deprecation_reason="セキュリティ上存在してはいけないAPIのためdeprecated")
    contenttypes = graphene.List(ContenttypeType, description="コンテンツタイプ一覧取得API")

    def resolve_users(self, info):
        return User.objects.all()

    def resolve_contenttypes(self, info):
        return ContentType.objects.all()

２種類のスキーマですが、出力される内容も異なっておりJSON形式で出力した場合、queryのdescription、型のdescription、deprecation_reason全て出力されていることに対し、graphql形式はdeprecation_reasonしか出力されていません。

そのため、今回のドキュメント化にはJSON形式のスキーマを利用します。

ドキュメントの出力

@2fd/graphdocを利用します。

$ npm install @2fd/graphdoc
$ graphdoc -s schema.json -o docs/

docs/配下にHTMLファイルが展開されます。そのままの状態では少し扱いづらいのでPagesで公開させます。(いつも通り)

.github/workflows/doc.yml 
name: doc

on:
  push:
    branches:
      - main


jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - uses: actions/setup-python@v3
      with:
        python-version: '3.10'
        cache: 'pip'
        cache-dependency-path: 'requirements.dev.txt'
    - uses: actions/setup-node@v2
      with:
        node-version: '16'
    - name: npm install
      run: |
        npm install -g @2fd/graphdoc
    - name: pip install
      run: |
        pip install -r requirements.txt
    - name: git init
      run: |
        git config --global user.name "${GITHUB_ACTOR}"
        git config --global user.email "${GITHUB_ACTOR}@users.noreply.github.com"
        git remote set-url origin https://x-access-token:${{ secrets.GITHUB_TOKEN }}@github.com/$GITHUB_REPOSITORY
    - name: generate document
      run: |
        graphdoc -s schema.json -o docs/ 
    - uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./docs
    - uses: actions/upload-artifact@v3
      with:
        name: schema
        path: |
          schema.json
          schema.graphql

GitLabの場合、GitHub Pagesの公開方法と異なり、CI内部でのディレクトリを指定すると自動でデプロイしてくれるのですがGitHubの場合、ブランチ上のdocs/もしくはroot dirにHTMLを置く必要があります。

その際、peaceiris/actions-gh-pages@v3 を使うと自動で指定したディレクトリをgh-pagesブランチにpushしデプロイを行なってくれるためとても便利です。
