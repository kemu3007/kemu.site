marked.jsとhighlight.jsでGitHub Flavored Markdownを扱う

GitHub上で扱っているMarkdownの正式名称はGitHub Flavored Markdownと言われ、デフォルトのMarkdownを拡張した独自記法になっています。

今回はhighlight.jsのGitHub cssを適用してMarkdownテキストをHTMLに変換します。

GitHub Flavored Markdown spec

今回使用したライブラリ

  • highlight.js
  • marked.js

導入

bash $ npm i highlight.js marked tipsのコードを実行する場合は crypto-es も同時にインストールしてください。

GFMをHTMLに変換する

MarkdownテキストをHTMLに変換し、コードブロックに対してhighlight.jsのcssがあたる様に設定します。

```ts import { marked } from 'marked'; import hljs from 'highlight.js';

const gfm_text = ""

marked.setOptions({ highlight: function (code, lang) { const language = hljs.getLanguage(lang) ? lang : 'plaintext'; return hljs.highlight(code, { language }).value; }, langPrefix: 'hljs language-', gfm: true, breaks: true, });

markd.parse(gfm_text) ```

highlight.jsのcss適用

style.scss

scss @import '~highlight.js/styles/github.css';

これによりMarkdownをGitHub上と同じように表示することができます。コードブロックのbackground-colorが存在していないなどがある場合は適宜cssを追加して適用してください。

scss :host ::ng-deep pre > code { background-color: #f6f8fa; border-radius: 10px; }

tips: 以下のような形でrendererを上書きしカスタマイズすることもできます。

```ts import { marked, Renderer } from 'marked'; import crypt from 'crypto-es'; import hljs from 'highlight.js';

const renderer = new Renderer(); /* ヘッダーにはテキストのhash値から生成したフラグメント, リンクをコピーするボタンを追加する / renderer.heading = (text, lebel) => { const hash = crypt.MD5(text); return `

${text}

; }; /** img要素にloading lazyを追加する */ renderer.image = (href) =>; /** リンク要素にbootstrap iconのリンクマークを付与する */ renderer.link = (href, title, text) =>${text}`; marked.setOptions({ renderer, highlight: function (code, lang) { const language = hljs.getLanguage(lang) ? lang : 'plaintext'; return hljs.highlight(code, { language }).value; }, langPrefix: 'hljs language-', gfm: true, breaks: true, });

```

これによりGFMをレンダリングすることができました。
