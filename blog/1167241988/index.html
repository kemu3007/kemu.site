GitHub PagesでデプロイしたSPAにOGPを表示させる(気合)

通常SPAをデプロイさせた場合、twitterなどでOGPが表示されません。 これはOGPを取得するbotがjsを実行しないこと、それとSPAのため該当するリンク先に静的なHTMLファイルが存在していないことが原因です。 そのため、通常はlambda等を利用しbot用のページ表示、それ以外のアクセスはS3に飛ばすといった手段が用いられます。 ただ今回はそこまで労力をかけずGitHub Pagesのみで完結させたいという思いがあったため解決方法を模索しました。 ### 解決方法 _ページがないならページを追加すればいいじゃない_ 今回は、angular.jsonのassetsに該当のリンク先ディレクトリ(今回であれば"src/blog")を追加し、ビルド時に保持させるようにしました。 ページの作成はデプロイ時にActionでPythonスクリプトを実行させ以下のような形でファイルを生成させています。 ``` src/blog/index.html src/blog/<:id>/index.html ``` 参照: https://github.com/kemu3007/portal/blob/master/create_pages.py やっている内容としてはベースとなるHTMLを先に用意し、含まれているtitle, image, descriptionの中身を該当ページで表示したいものに修正、ページのリンクと一致するディレクトリにHTMLファイルを保存する。という形になります。 これで直接リンクを踏んだ際にSPAを経由せず静的なOGPのみが含まれたHTMLに遷移するのでこのページ内でAngularのアプリケーションが存在しているルートリンクに遷移するようにし、現在のパスをクエリパラメータで渡し、内部リンクで遷移するようにしています。 ※ app-componentでこの処理を行うとrouter-outlet内部ではないため `route.snapshot.queryParams` が空になってしまうので注意が必要です。 bot: リンク -> `OGPのみのHTML` 閲覧者: リンク -> `OGPのみのHTML` -> `Angular Application` 一度アプリケーションに遷移すれば以降は内部リンクで全て処理されるため遷移に関する問題は発生しません。 ### 感想 思いついてみれば簡単で単純なやり方なのですが少し時間がかかってしまいました。 気合いでOGPを解決した形なので仕事でこれをやるかと言われればやらないんですがまあ個人ブログだしええかという精神です。 SEO対策も少し齧っているくらいで全然わからんなという感じなので、もっといい方法があればコメントで教えてください。
