AWS コンソールへのサインインが行われた場合に Slack 通知を行う

発行日時: 2023.07.23 04:53

最終更新日: 2023.07.23 04:53

不正アクセス対策の一環とし、ConsoleLogin イベントの通知を行います。
ConsoleLogin イベントは以前まで us-east-1 リージョンでのみ発生し、EventBridge を利用して簡単に検出ができていたのですが、現時点ではサインイン時に利用されたリージョンにて発生するイベントとなっているため、少し凝った方法を利用する必要があります。

方法として大きく二つ考えることが可能です。

  1. CloudTrail にてマルチリージョントレイルを作成し、CloudWatch Logs に出力。サブスクリプションフィルターから Lambda 関数を呼び出し通知処理を行う
  2. EventBridge ルールを各リージョンで作成し、通知処理をトリガーする

今回は私用アカウントということもあり、手順およびコストを最小化したいものとなりますため、新機能である AWS User Notifications [1] を利用しました。

AWS User Notifications を使う

AWS User Notifications を利用する場合、通知文章を変更することが現時点でできません。
とりあえず攻撃されてないかノリでわかればいいや、という人におすすめです。

設定内容

サービス名 

AWS Console Sign-in

イベントタイプ 

AWS Console Sign In via CloudTrail

イベントパターン JSON 

{
  "detail": {
      "eventName": ["ConsoleLogin"]
  }
}

対象リージョン 

ap-northeast-1
ap-northeast-2
ap-northeast-3
ap-south-1
ap-southeast-1
ap-southeast-2
ca-central-1
eu-central-1
eu-north-1
eu-west-1
eu-west-2
eu-west-3
sa-east-1
us-east-1
us-east-2
us-west-1
us-west-2

上記設定を行うことで各リージョンで自動的に EventBridge ルール作成が行われます。
たったこれだけで ConsoleLogin 検知が可能...便利だ。。。

通知内容

以下の内容は通知より確認が可能です。
Region / Event ID があるので CloudTrail ログも検索がしやすく情報が薄いということはなさそうですね。

  • Region
  • Account ID
  • User identity
  • User agent
  • Login to
  • Login attempt
  • Event ID
  • Event time
